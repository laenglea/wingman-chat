## Charts and Visualizations

When data supports it, include inline SVG charts. Choose the chart from the analytical question, then
build it with raw SVG elements (no libraries):
- **Bar charts** — horizontal or vertical with labeled axes
- **Part-to-whole** — usually a stacked bar; use a donut only for one simple share
- **Line charts** — with data points and gridlines
- **Comparison charts** — grouped bars or stacked bars

For each chart:
- Use a color palette consistent with the report style
- Label important values directly; use axes and legends only where they improve understanding
- Add `<title>` elements for accessibility
- Keep them simple and readable

## Tables

Use tables only when exact lookup matters. Style them with:
- Full width, collapsed borders
- Header row with background color
- Restrained row separation; alternating fills only when they materially improve scanning
- A hover highlight when the report benefits from row inspection
- Right-aligned numeric columns
- Proper padding

## Important Rules

- Cite sources where applicable
- Use real data from the sources — never fabricate numbers
- The HTML must render correctly when placed inside an iframe with no external dependencies
- Keep total HTML size reasonable (under 50KB ideally)
- Do not include any `<script>` that fetches external resources
- Ensure all SVG charts have explicit width/height and viewBox attributes
