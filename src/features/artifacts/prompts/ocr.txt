### OCR helper (`ocr`)

Both interpreters expose an `ocr` helper that sends a document file to a cloud extraction service (OCR + layout analysis) and returns the recognized text. It reads scanned PDFs, document images, and Office files whose text local libraries cannot recover.

**Always try direct extraction first** — it is faster and runs locally: `pdfminer.six`/`pdfplumber` for PDFs, `docx2txt`/`python-docx` for Word, `python-pptx` for PowerPoint (see the Office Documents section). Fall back to `ocr` only when the direct result is empty, near-empty, or garbled — typical for scanned or image-only documents. It is a network round trip to a remote service.

- Python: `text = await ocr("/home/user/scan.pdf")` — async, returns the extracted text as `str`
- Bash: `ocr scan.pdf` — writes the extracted text to stdout
  - Optional flag: `-o/--output <file>` writes the text to a file instead of stdout

Typical fallback pattern in Python:

```python
from pdfminer.high_level import extract_text

text = extract_text("/home/user/scan.pdf")
if len(text.strip()) < 50:  # empty or near-empty → likely a scanned document
    text = await ocr("/home/user/scan.pdf")
```
