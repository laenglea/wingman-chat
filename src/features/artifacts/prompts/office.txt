## Office Documents (DOCX, PPTX, XLSX, PDF, Email)

Uploaded `.docx`, `.xlsx`, `.pptx`, `.pdf`, `.msg`, and `.eml` files are preserved verbatim under `/home/user/`. Read or extract their contents with the Python libraries below — don't ask the user to re-upload as Markdown.

To *create* a polished Word doc, slide deck, spreadsheet, or PDF, use the **Studio** capability: its `docx`, `pptx`, `xlsx`, and `pdf` skills carry the full build recipes — `read_skill` the matching one before building. Write any output file under `/home/user/` with the exact extension the user expects so it syncs back as an artifact; don't paste the generated bytes into chat.

Available libraries (all preloaded, no network install needed):
- DOCX: `python-docx` (import `docx`), `docx2txt` (extract text)
- PPTX: `python-pptx` (import `pptx`)
- XLSX: `openpyxl`, `xlsxwriter`, `pandas` (`pd.read_excel` / `df.to_excel`)
- PDF: `reportlab` (create), `pypdf` (merge, split, watermark, rotate, quick text), `pdfminer.six` (import `pdfminer`; high-fidelity text & layout extraction — prefer over `pypdf` for prose or columns), `pdfplumber` (structured **table** extraction — `page.extract_tables()`, and `page.extract_text()`)
- Email: `extract-msg` (import `extract_msg`) for `.msg`; stdlib `email`/`email.parser` for `.eml`
- Charts: `matplotlib`, `seaborn`, `plotly` (write PNG/SVG and embed)

Sandbox notes (these traps apply even outside Studio):
- There is no LibreOffice, Pandoc, `markitdown`, `pdftoppm`, or `recalc.py` here, so any skill or habit that shells out to one does not apply. Don't render slides/docs to images or recalculate formulas externally.
- `openpyxl` does not evaluate formulas: reading a workbook back with `data_only=True` returns `None` for formula cells. That's expected — Excel/Sheets/LibreOffice recalculates when the file is opened. Never "fix" it by computing the value in Python and hardcoding it.
- To edit an existing file, open it with the matching library (`Document(path)` / `Presentation(path)` / `load_workbook(path)`), change only what's needed, re-save, and match the file's existing styling and conventions — don't impose new defaults.
- Smart quotes ("don't") are fine — pass the Unicode characters directly in Python strings. Never paste Unicode bullets (`•`, `·`) into Office documents; use the library's list/numbering APIs.

### Pull tables out of a PDF

`pdfplumber` returns rows as lists; feed them straight into pandas:

```python
import pdfplumber, pandas as pd

with pdfplumber.open("/home/user/in.pdf") as pdf:
    for page in pdf.pages:
        for table in page.extract_tables():
            df = pd.DataFrame(table[1:], columns=table[0])
            # … e.g. df.to_excel("/home/user/out.xlsx", index=False)
```

`extract_tables()` detects tables by their **ruling lines**, so it returns `[]` for borderless or whitespace-separated tables. In that case pass a text-based strategy: `page.extract_tables({"vertical_strategy": "text", "horizontal_strategy": "text"})`. (`extract_text()` still returns the content either way — fall back to parsing that if detection fails.) For prose or multi-column layouts, `pdfminer.high_level.extract_text(path)` is far more faithful than `pypdf`.

### Email (.msg / .eml)

Use `extract_msg` for Outlook `.msg`; the stdlib `email` package for `.eml`.

```python
# .msg
import extract_msg
m = extract_msg.openMsg("/home/user/email.msg")
print(m.sender, m.to, m.subject, m.body)
for att in m.attachments:
    att.save(customPath="/home/user/")
```

```python
# .eml
from email import policy
from email.parser import BytesParser
m = BytesParser(policy=policy.default).parse(open("/home/user/message.eml", "rb"))
body = m.get_body(preferencelist=("plain", "html"))
print(m["from"], m["subject"], body.get_content() if body else "")
```

### Chart → Office embed

For charts inside a DOCX/PPTX/PDF, render with matplotlib (or seaborn/plotly→png) to `/home/user/chart.png`, then insert with the library's image API (`doc.add_picture`, `slide.shapes.add_picture`, `Image(...)`). Use a non-interactive backend:

```python
import matplotlib
matplotlib.use("Agg")
import matplotlib.pyplot as plt

fig, ax = plt.subplots(figsize=(8, 4), dpi=150)
ax.plot([1, 2, 3], [10, 25, 40])
fig.savefig("/home/user/chart.png", bbox_inches="tight")
plt.close(fig)
```
