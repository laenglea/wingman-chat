### PDF rasterization helper (`rasterize_pdf`)

Both the Python and JavaScript interpreters expose a helper that renders PDF pages to PNG images. The sandbox has no in-process PDF rasterizer — `pypdfium2`, PyMuPDF, and poppler are all unavailable, so `pdfplumber`'s `.to_image()` raises `ModuleNotFoundError: No module named 'pypdfium2'`. This helper is the only way to turn a PDF page into an image: for visual inspection, overlaying form-field boxes, OCR of a specific region, or feeding a page to `vision`.

- Python: `rasterize_pdf(path, scale=2.0, pages=None)` renders the requested pages (a page number, a list, or None for all) and returns the written PNG paths, e.g. `["report-1.png", "report-2.png"]`. It is async — `await` it (or call it at the top level of your code).
- JavaScript: `rasterizePdf(path, { scale, pages })` returns the written paths through the `vfs`.

`scale` controls resolution: 1.0 ≈ 72 DPI, 2.0 ≈ 144 DPI. Higher scale means larger images, so the pixel coordinates of anything you draw scale with it. Render only the pages you need — each page is a full raster. For plain text or tables, prefer `pdfplumber`/`pypdf` (vector extraction, no raster); reach for `rasterize_pdf` only when you genuinely need pixels.
