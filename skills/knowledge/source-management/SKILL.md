# Source Management

Use this as supporting guidance for search and digest tasks.

## Detect Sources

Inspect the available tool list and infer which source types can actually be queried. Common source classes include chat, email, cloud files, local files, project trackers, CRM, databases, and knowledge bases, but only use classes that are backed by available tools.

## Source Priority

- Decision questions: meeting notes, thread conclusions, approval records, then follow-up announcements.
- Status questions: project/task systems and recent discussions.
- Document questions: file search and knowledge bases before conversation search.
- Policy/factual questions: official docs and maintained knowledge bases before informal discussion.
- People/context questions: task ownership, document authorship, and recent discussion participants.

Priority affects ranking, not coverage. Search lower-priority sources when they may contain missing context.

## Missing Sources

If a needed source is unavailable, state that directly and continue with available sources. Suggest the smallest useful next source to connect or provide, such as a specific file, export, workspace, or project tracker.

## Failure Handling

If one source fails, continue with other sources and disclose the failure in the answer. Do not present partial coverage as complete.
